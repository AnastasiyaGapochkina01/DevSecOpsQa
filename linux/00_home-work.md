### Задача 1
1) В домашней директории своего пользователя создать директорию `practice`; в `practice` создать пустой файл `system-info`
2) Записать с помощью пайплайнов и перенаправлений в файл `system-info` следующую информацию
- текущую дату и время
- размер диска, смонтированного под корневую директорию
- общее количество оперативной памяти
- количество ядер CPU
- текущий load average
- 3 директории в системе, которые занимают больше всего места
3) Создать пользователя `devops` и группу `external`
- добавить пользователя `devops` в группу `external`
- создать в директории `/opt` директорию `projects`; в директории `projects` создать файлы
    - `app.py`
    - `run.sh`
    - `main.go`
- рекурсивно назначить владельцем директории `projects` пользователя `devops`, а группой - `adm`
- сделать файл `run.sh` исполняемым для всех
- назначить права на файл `app.py` так, чтобы владелец имел полные права, групп могла читать и изменять, а все остальные не имели никаких прав
- назначить права на файл `main.go` так, чтобы владелец мог читать и изменять файл, группа могла читать и выполнять, а все остальные - только читать 
4) Создать резервную копию директории (сжатый архив) `projects` в директории `/opt/backups`
5) Вывести все сетевые интерфейсы в статусе UP; отфильтровать вывод так, чтобы получить только их имена и ipv4 адреса; например
```bash
eth0 10.129.0.36
```
6) Выполнить команду `bash -c 'sleep 300 & exec sleep 400' &`; далее:
- вывести дерево процессов и определить какой из запущенных процессов sleep родитель, а какой потомок
- найти PID процессов sleep
- найти PPID процессов sleep
7) Выполнить команду `yes > /dev/null &`; далее
- в файл `top_cpu` записать 3 процесса с максимальным потреблением cpu (использовать пайплайны и перенаправления)
- в файл `top_mem` записать 3 процесса с максимальным потреблением memory (использовать пайплайны и перенаправления)
- убить процесс `yes > /dev/null &`
8) Установить на ВМ nginx и memcached
9) В файл `exernal_services` записать информацию (использовать пайплайны и перенаправления):
- запущены ли сервисы nginx и memcached
- добавлены ли они в автозагрузку

> [!WARNING]  
> Формат файла `exernal_services` должен быть таким
> ```
> ===> scout-backups 2025-12-01_03:33:18 <===
> memcached active and enabled
> nginx active and enabled
> ```
> где `scout-backup` это имя **ВАШЕЙ** ВМ; `2025-12-01_03:33:18` - текущая дата и время

10) Пользователю devops, которого мы создали в задаче 3 дать права на выполнение команд systemctl stop/start/restart/status от имени root без пароля
